1) Основві типи віджетів, які ми використовували: TextView, Button, Timer. Також мені відомі наступні віджети: RadioButton, CheckBox, DatePicker, CalendarView, ZoomControls, ImageView, ScrollView, ProgressBar, RatingBar, тощо.
2) Щоб опрацювати натиснення кнопки необхідно прописати властивість кнопки android:onClick і додати функцію, яка буде при цьому спрацьовувати
3) Для TextView існують наступні атрибути: android:gravity, android:letterSpacing, android:textFontWeight, android:textSize, android:textStyle. З методів існують setText, setTextColor.

1) ProgressBar:
- на формі додається елемент ProgressBar
- на момент ініціалізації бара встановлюється максимальне значення методом: progressBar.setMax(MAX);
- для зміни значення на барі використовуємо метод: progressBar.setProgress(timeLeft);
- в нашому випадку зміна в прогрес барі відбувається кожної секунди так як рухається таймер.
2) CheckBox, RadioButton
для CheckBox ми маємо метод isChecked(), якщо хочемо перевірити чи включений об'єкт. Якщо ми хочемо обробити подію натискання, то можемо скористатись подією onCheckedChanged або onClick і прописати логіку в залежності від стану об'єкту.
Для обробки натискання RadioButton існує подія onRadioButtonClicked, де ми можемо отримати доступ до вибраного елемента за допомогою view.getId()
3) Spinner
Для обробки вибору можна використовувати подію onItemSelected, для зчитування обраного елемента використовують spinner.getSelectedItem().toString()


1. Для чого використовуються об’єкти intent? Їх типи.
Intent використовується для запуску іншої Activity, виклику служб, розсилки повідомлень. Типи intent: явні і неявні.
Явний інтент вказує, що потрібно запустити явно. Неявні об’єкти Intent не містять імені конкретного компонента. Замість 
цього, вони визначають дію, яку потрібно виконати. Система сама вирішує, для 
яких компонентів спрямовано Intent шляхом порівняння змісту об’єкта Intent з 
фільтрами Intent, які визначені в файлах маніфеста інших додатків.
2. Призначення об’єктів bundle.
Системні дані, що використовуються системою для відновлення попереднього стану зберігаються у вигляді набору пар «ключ-значення» та 
містяться у об’єкті Bundle.
3. Яким чином реалізуються різні екрани для різних орієнтацій?
У проекті створюється кілька layout-файлів для однієї activity з однаковим ім'ям, але розташованих у різних папках у залежності від орієнтації.
